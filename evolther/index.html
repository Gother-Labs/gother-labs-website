Evölther

An operating system for governed improvement loops.

Offline-first, auditable, and built for real constraints.

What we choose to show.

We do not need to expose every internal detail to make the system legible. But we do want to show enough to make the mechanism understandable: bounded edits, explicit scoring, artifacts, and a verdict that can be inspected. 

$ evolther run domain=energy_kernel

[00] freeze baseline ............. ok
[01] lock evaluation contract .... ok
[02] materialize candidate ws .... ok
[03] patch EVOLVE block .......... applied
[04] enforce hard constraints .... passed
[05] evaluate holdout ............ delta +1.8%
[06] replay stress scenario ...... passed
[07] export trace ................ telemetry.jsonl
[08] verdict ..................... proceed

Editing surface

Explicit EVOLVE regions instead of arbitrary rewrites.

Run artifacts

`config.yaml`, `telemetry.jsonl`, finalists, and verdicts.

Outcome discipline

No-signal is valid if the diagnosis is clear and auditable.

We are not interested in making optimization look magical. We are interested in making it governed.
